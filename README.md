## 这是什么

Legal Skills 是一套中文法律工作 Agent Skill 集合。它把法律工作从“让模型直接回答”改造成“让 Agent 按流程推进”：先识别任务类型，再读取材料、复查来源、校验法规、暴露缺口、生成草稿，并在正式交付前做出稿审查。

这个项目的核心不是某一个万能提示词，而是一组可组合的法律工作流：

- `法律工作总控` 负责统一入口、语义路由和共享规则。
- 各业务子 Skill 负责具体专业流程，例如合同审查、刑事辩护、民事诉讼、破产管理、广告合规等。
- 文书与导出 Skill 负责正式交付前的结构检查、身份占位、路径校验和 DOCX 导出链路。

它适合律师、法务、法律科技开发者和 AI Agent 使用者，用来搭建本地法律工作台、法律文书草稿系统、案件材料处理流程或专项合规审查助手。

> **重要提示：本项目不提供法律意见。**
> 所有输出都应被视为供律师、法务或合格专业人士复核的工作草稿。使用者必须自行核验事实材料、法律法规、案例时效、授权范围、诉讼期限和最终交付内容。

## 为什么做这个

法律工作很少只是“问一个问题、给一个答案”。真实工作通常包含：

- 材料是否完整、是否读全、是否存在 OCR 错误；
- 引用的法规是否现行有效；
- 案件事实、金额、日期、主体和证据之间是否能相互印证；
- 输出给客户、法院、仲裁委、交易相对方或内部团队时，口径是否不同；
- 正式文书是否需要 Word 交付、签章信息、目录、表格、附件和版本记录；
- 哪些内容可以进入业务文件区，哪些只能留在内部系统记录。

Legal Skills 的设计目标是把这些隐形步骤显式化，让 Agent 不只是“会写”，还要“会问、会查、会复核、会暴露不确定性、会在交付前刹车”。

## 能做什么

| 模块 | 主要用途 |
|---|---|
| **法律工作总控** | 统一入口、语义路由、事项隔离、来源边界、材料读取复查、法规/Wiki 校验、输出头规则 |
| **法律咨询助手** | 微信式客户咨询快速回复，适合初步沟通、聚焦追问和简短建议 |
| **民事诉讼** | 初步法律分析、法规案例检索、调查取证、诉讼文书、立案、庭前准备、庭审、调解与结案 |
| **刑事辩护** | 案件承接、侦查阶段、审查起诉、一审、二审、简易速裁、未成年人、死刑和特殊程序 |
| **劳动争议** | 劳动争议诉讼、仲裁程序管理、证据体系、劳动关系认定、经济补偿和加班费测算 |
| **破产程序** | 破产申请与受理、管理人工作、财产调查、债权申报、债权人会议、重整、和解、清算 |
| **合同与交易** | 合同起草、合同审查、续约提醒、版本比对、合同偏好学习和最终版归档 |
| **专项合规** | 产品法务、用人单位劳动合规、广告合规、预包装食品标签合规、监管动态监测 |
| **法律文书** | 出稿前审查、模板与 DOCX 导出、审理报告、民事判决书、破产文书和服务建议书 |
| **工具型 Skill** | 法条转 Markdown、微信文章转 Markdown、法律文章去 AI 味、诉讼可视化、诉讼分析 |

## 工作方式

一个典型任务会经历这些步骤：

```text
用户提出法律任务或提供材料
        │
        ▼
法律工作总控识别任务类型
        │
        ▼
路由到合同、诉讼、刑辩、劳动、破产、合规等子 Skill
        │
        ▼
读取材料、网页、图片或表格，并输出读取复查摘要
        │
        ▼
需要法规或案例时进行现行有效性和来源校验
        │
        ▼
生成分析、清单、提纲、文书草稿或可视化图表
        │
        ▼
正式 Word 交付前进入出稿前审查
        │
        ▼
通过后再导出 DOCX 或给出最终交付文本
```

其中，`法律工作总控` 是整个系统的入口。用户可以直接指定子 Skill，也可以只描述任务，由总控根据语义路由。

## 快速开始

### 方式一：作为普通 skills 目录安装

```bash
git clone https://github.com/pa1nrui1/legal-skills.git
mkdir -p ~/.codex/skills
rsync -a legal-skills/skills/legal/ ~/.codex/skills/legal/
```

安装后，在 Codex 或其他支持本地 skills 的 Agent 中直接调用：

```text
法律工作总控 帮我判断这个任务应该走哪个法律工作流。
```

```text
合同审查 审核这份合同，按风险等级输出修改建议。
```

```text
法律咨询助手 帮我回复这个客户咨询，保持微信式简短自然。
```

```text
民事一审诉讼 读取这个案件材料，先做读取复查摘要，不要直接生成法律意见。
```

### 方式二：作为 Codex plugin 安装

```bash
codex plugin marketplace add pa1nrui1/legal-skills --sparse .codex-plugin --sparse skills
```

## 目录结构

```text
skills/legal/
  法律工作总控/
    SKILL.md
    references/
    scripts/
  合同审查/
  合同起草/
  民事一审诉讼/
  刑事辩护总调度/
  法律文书出稿前审查/
  法律文书模板与导出/
  ...
```

每个子目录通常包含：

- `SKILL.md`：触发条件、适用范围、执行顺序和禁止事项。
- `references/`：流程、方法论、检查表、规则和专项说明。
- `templates/`：文书模板、报告框架、清单和交付物样式。
- `scripts/`：可选的本地自动化脚本。

## 主要 Skill 入口

| 入口 | 适合什么时候用 |

## 本地配置

本仓库已删除作者个人执业身份、联系方式和作者本机路径。用于真实业务前，你需要按自己的环境配置：

- 律师姓名、律所名称、地址、电话、邮箱等身份占位；
- 工作区、客户台账、系统记录、业务文件区等本地保存规则；
- 法规检索、案例库、OCR、飞书或其他外部服务凭证；
- 是否允许 Agent 写入系统记录、日历、云文档或其他外部系统。

建议把真实业务配置保存在本机私有文件或环境变量中，不要提交到公开仓库。

## 安全边界

Legal Skills 默认强调以下边界：

- 不把模型记忆包装成文件事实、现行法规或已核验案例。
- 不在未读取材料、未完成复查、未校验法规时生成正式法律意见。
- 不静默写入台账、系统记录、飞书文档或日历。
- 不把客户材料、案卷、OCR 中间结果、API key 或商业数据库凭证提交到仓库。
- 不替代律师、法务或专业人士对事实、证据、法律适用和最终文本的判断。

## 开源范围

本仓库发布的是 skill 规则、流程、模板和脚本，不包含：

- 作者个人身份信息；
- 作者本机路径；
- 真实客户材料；
- 业务台账或系统记录；
- 飞书文档；
- 私有数据库凭证；
- 迁移备份目录。

开源整理时已将文书模板中的身份信息改为通用占位符，并将外部服务凭证改为显式环境变量配置。

## 适配其他 Agent

这些 Skill 本质上是 Markdown 工作流说明和少量脚本。即使不使用 Codex，也可以把 `skills/legal` 目录复制到其他支持本地规则、工作流或知识文件的 Agent 工具中，再根据该工具的加载机制做适配。

适配时建议保留：

- `SKILL.md` 的触发条件和执行顺序；
- `法律工作总控/references/` 中的共享协议；
- `templates/` 中的文书结构；
- 出稿前审查和文书导出的质量门。
